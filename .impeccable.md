## Design Context

### Users
Micah Boswell — senior UX/product designer and AT&T Experience Lead. Uses this as a personal observatory for monitoring LLM API costs, quality, and routing decisions across his own projects. Power user who reads every metric. Context: dark office, technical work, late-evening review sessions.

### Brand Personality
"precise · warm · analytical" — Claude's design language applied to a dark observability dashboard. Not cold/clinical like traditional devops tools. Not maximalist. Quietly confident.

### Aesthetic Direction
Claude's warm terracotta design language adapted for dark mode:
- Primary accent: #D97757 (Claude's characteristic terracotta)
- Secondary palette: #C9966B amber, #A89276 warm stone, #8BA89C sage, #9EA87A olive
- Warm dark backgrounds: #211C18 / #19150F (warm-tinted ink, not cool)
- Tooltip cards: #211C18 gradient with terracotta border
- Data visualization: radial gradient circles with glow on hover, warm-tinted chart areas
- Typography: editorial weight hierarchy — mix of weight and size for hierarchy, not all-caps everywhere
- Grid lines: barely visible (rgba 3-4%)
- No border-left stripes (banned), no gradient text (banned)
- Warm neutral text: rgba(200,185,165,x) scale rather than pure white/gray

### Design Principles
1. Warm over cool — every color tints toward #D97757, never toward cyan/blue
2. Data first — chrome serves data, never competes with it
3. Editorial headers — title in weight/size, subline in muted caps, not the other way around
4. Real data or honest fallback — demo badge when showing synthetic data, no-scores badge when real data lacks quality
5. Hover reveals, not always-on — labels, detail rings appear only on hover to keep the chart clean at rest
