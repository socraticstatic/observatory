## Design Context

### Users
Micah Boswell — senior UX/product designer and AT&T Experience Lead. Uses this as a personal observatory for monitoring LLM API costs, quality, and routing decisions across his own projects. Power user who reads every metric. Context: dark office, technical work, late-evening review sessions.

### Brand Personality
"precise · cold · analytical" — Misty Metallic engineering aesthetic applied to a dark observability dashboard. Not warm/inviting. Not devops brutalist. Engineered calm.

### Aesthetic Direction — Cold Steel (implemented)
Misty Metallic cold monochrome system with a single cool teal accent:
- Backgrounds: --ink #11171B / --ink-2 #0B1014 (cool-tinted near-black, not warm)
- Surface scale: --slate #232A2E → --graphite #4A5358 → --steel #8A9297 → --fog #C8CED1 → --mist #E9ECEC
- Primary accent: --accent #6FA8B3 (cool teal)
- Secondary accent: --accent-2 #9BC4CC (lighter teal)
- Semantic: --warn #C9966B · --bad #B86B6B · --good #7CA893
- Data viz palette (CSS vars): --dv-teal / --dv-sage / --dv-rust / --dv-olive / --dv-stone / --dv-amber
- Typography: Geist (via __nextjs-Geist) + JetBrains Mono for numerics. 4px corner radius everywhere.
- No border-left stripes (banned), no gradient text (banned), no warm tints

### Design Principles
1. Cold over warm — every color tints toward #6FA8B3 teal, never toward orange/terracotta
2. Data first — chrome serves data, never competes with it
3. Engineered hierarchy — label in 10px uppercase tracking, value in Mono weight/size. Not inverted.
4. Real data or honest empty state — loading guards on every chart, no fake fallback data
5. Hover reveals, not always-on — tooltips and detail labels appear on hover only
